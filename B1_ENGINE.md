## 1. Build Scope

B1 is the Contradiction Detection Engine — the foundation of the entire Nine-Brain system. All other brains (B2-B8) operate downstream of B1's findings. B9 validates B1's output but does not vote on it.

**What B1 Does:** Reads every document, transcript, and chat log. Extracts every proposition (claim, assertion, statement). Pairs propositions that contradict each other within the same subject category. Assigns severity. Anchors each contradiction to person + page/line + statute.

**What B1 Does NOT Do:** Interpret intent, assign guilt, or make legal conclusions. B1 finds contradictions. B7 maps them to law. The human investigator draws conclusions.

## 4. Statement Type Weighting

Different types of statements carry different evidentiary weight. B1 uses this weighting in severity calculation.

| Type | Weight | Description |
|------|--------|-------------|
| Sworn testimony | 5x | Statements made under oath in court or affidavit |
| Contemporaneous note | 4x | Notes made at the time of events |
| Email (internal) | 3x | Internal communications not intended for external view |
| Email (external) | 3x | Communications with third parties |
| Contract clause | 4x | Formal contractual provisions |
| Invoice/financial | 3x | Financial records |
| WhatsApp/chat | 2x | Informal messaging |
| Verbal (witnessed) | 2x | Spoken statements with witness |
| Verbal (unwitnessed) | 1x | Hearsay or unverified claims |

**Rule:** A contradiction between two sworn statements (5x vs 5x) scores CRITICAL. A contradiction between sworn testimony and an internal email (5x vs 3x) scores HIGH. A contradiction between chat messages (2x vs 2x) scores MODERATE unless corroborated by other brains.

## 5. Severity Levels

| Level | Score Range | Meaning | Action Required |
|-------|------------|---------|----------------|
| **CRITICAL** | 8-10 | Direct contradiction between sworn statements or between sworn statement and sealed document | Must include in sealed report. Flagged for immediate investigator attention. |
| **HIGH** | 6-7 | Contradiction between sworn statement and contemporaneous evidence, or between contract and conduct | Must include in sealed report. Flagged for investigator review. |
| **MODERATE** | 4-5 | Contradiction between informal communications, or single-instance inconsistency | Included in sealed report. Noted for pattern analysis. |
| **LOW** | 2-3 | Minor inconsistency, typographical error, or ambiguous statement | Logged. Included only if pattern emerges across multiple instances. |
| **INSUFFICIENT** | 0-1 | Cannot determine contradiction due to missing context | Flagged for B9 coverage gap analysis. |

## 6. Algorithm

```
INPUT: evidence_bundle (list of EvidenceDocument)
OUTPUT: contradiction_list (list of Contradiction, severity-sorted)

1. EXTRACT ALL PROPOSITIONS
   For each document in evidence_bundle:
     For each page/line:
       Extract all claims, assertions, statements
       Record: text, person, page, line, statement_type, timestamp

2. CATEGORIZE PROPOSITIONS
   For each proposition:
     Assign to one of 7 categories (Goodwill, Contract, Signature, Section 12B, Compensation, Perjury, Coercion)
     If unassignable → flag for B9 review

3. PAIR CONTRADICTIONS
   For each category:
     For each proposition pair (A, B) in category:
       If A and B are semantically opposite:
         Calculate temporal gap (|timestamp_A - timestamp_B|)
         Calculate severity = f(statement_type_A, statement_type_B, temporal_gap, category)
         Create Contradiction object with anchors

4. MULTI-PASS SCANNING
   Pass 1: Direct contradictions (A says X, B says not-X)
   Pass 2: Implied contradictions (A says X, B's actions assume not-X)
   Pass 3: Temporal contradictions (A said X at t1, B said X at t2, but evidence shows X changed)
   Pass 4: Cross-document judicial (sworn testimony vs. sealed document) — v5.2.8 OJRS-enabled

5. CROSS-BRAIN VALIDATION
   Send contradictions to B2-B8 for independent verification
   Require: B1 flagged + 2+ other brains confirm
   If quorum not met → mark INSUFFICIENT or INDETERMINATE_DUE_TO_CONCEALMENT

6. SORT AND OUTPUT
   Sort by severity (CRITICAL → HIGH → MODERATE → LOW)
   Group by category
   Generate anchor list (person + page/line + statute for each)
   Return sealed contradiction_list
```

## 7. Output Format

### Contradiction Object

```kotlin
data class Contradiction(
    val id: String,                    // Unique ID: C-{category}-{number}
    val category: ContradictionCategory, // One of 7 constitutional categories
    val type: ContradictionType,        // One of 16 types
    val severity: Severity,             // CRITICAL / HIGH / MODERATE / LOW / INSUFFICIENT
    val claimA: Claim,                  // First proposition
    val claimB: Claim,                  // Second proposition
    val temporalGapDays: Int,           // Days between claims
    val consciousnessOfGuilt: Boolean,  // True if gap > 730 days
    val anchorPerson: String,           // Person responsible
    val anchorPage: Int,                // Document page number
    val anchorLine: Int,                // Line number on page
    val anchorStatute: String,          // Applicable legal statute
    val verifiedBy: List<Brain>,        // Brains that confirmed (B1 + 2+ others)
    val sealHash: String                // SHA-256 of this contradiction record
)

data class Claim(
    val text: String,                   // The exact text of the claim
    val person: String,                 // Who made the claim
    val document: String,               // Document name
    val page: Int,                      // Page number
    val line: Int,                      // Line number
    val statementType: StatementType,   // Weighted type
    val timestamp: Instant              // When the claim was made
)
```

## 8. Integration Rules

### Upstream (What Feeds Into B1)
- `ForensicService.ingest()` — all evidence documents
- `MediaIngestor.kt` — photos, audio, video (transcribed before entering B1)
- `EntityExtractor.kt` (B2) — person/entity identification (feeds into proposition attribution)
- `JudicialRetrievalService.kt` (B7) — **v5.2.8: online judicial propositions from SAFLII/PACER/BAILII**

### Downstream (What B1 Feeds)
- `NineBrainEngine.kt` — B1 output goes to B2-B8 for validation
- `ReportGenerator.kt` — contradictions included in sealed forensic report
- `BehavioralBrain.kt` (B4) — contradiction patterns feed behavioral analysis
- `TimelineBrain.kt` (B5) — temporal gaps feed timeline reconstruction
- `LegalMapper.kt` (B7) — contradictions mapped to applicable statutes

### B9 Red-Team Testing
- B9 deliberately creates fake evidence with known contradictions
- Tests whether B1 finds them
- If B1 misses a known contradiction → calibration alert
- If B1 flags a false contradiction → sensitivity adjustment
- B9 reports: "B1 found {found}/{total} planted contradictions. Sensitivity: {score}"
