## Executive Summary

The Verum Omnis Identity & Trust System (VITS) is a five-component framework that binds human identity, device provenance, metadata integrity, document authenticity, and case-specific identity verification into a single, court-admissible chain of custody.

Built for the Constitution v6.0 Final and the Nine-Brain Architecture, VITS ensures that every document, every report, and every legal filing generated by the Verum Omnis platform can be traced back to a verified person, a specific device, and an immutable timestamp — creating a forensic trail that meets Constitutional Court standards.

## Component 2: DeviceIdentity

Creates an immutable hardware fingerprint that anchors every action to a specific physical device.

### Device Hash

```
SHA-512(device_serial : device_model : device_manufacturer)
```

Example: Samsung SM-A366B with IMEI 354601080768xxx → 128-character hex hash uniquely identifying that device across the entire Verum ecosystem.

### Chain of Custody Anchor

```
SHA-512(device_hash : user_profile_id : case_id : UTC_timestamp)
```

Binds four elements into a single forensic anchor. Any change to any element produces a completely different hash — tampering immediately detectable.

### Fields

| Field | Description |
|-------|-------------|
| device_serial | IMEI (mobile) or serial number (desktop) |
| device_model | e.g., SM-A366B, iPhone15,2 |
| device_manufacturer | e.g., Samsung, Apple |
| os_version | e.g., Android 14, iOS 17 |
| device_hash | SHA-512 fingerprint (auto-computed) |
| first_registered | ISO timestamp of first registration |
| last_seen | ISO timestamp of most recent activity |

## Component 3: MetadataFraudDetector

Analyzes document and image metadata to detect tampering, manipulation, and provenance fraud. Five separate checks producing an aggregate risk score from 0 (clean) to 100 (definitely fraudulent).

### Fraud Checks

| Check | What It Detects | Risk Points |
|-------|----------------|-------------|
| CREATOR_TOOL | Photoshop, GIMP, Canva, etc. | 40 (red flag) |
| DATE_CONSISTENCY | Modified before created (impossible) | 35 (red flag) |
| CAMERA_METADATA | Missing camera data on claimed photos | 20 (warning) |
| SOFTWARE_HISTORY | Multiple manipulation tools used | 30 (red flag) |
| HASH_INTEGRITY | SHA-512 of file for later comparison | 0 or 10 |
| METADATA_ABSENCE | No metadata at all (deliberate stripping) | 15 (warning) |

### Risk Scoring

| Score | Level | Action |
|-------|-------|--------|
| 0 | CLEAN | Document passes all checks |
| 1–19 | LOW | Minor anomaly — note in report |
| 20–39 | MEDIUM | Flag for reviewer attention |
| 40–69 | HIGH | Strong evidence of tampering |
| 70+ | CRITICAL | Reject document, alert user |

### Red-Flag Creator Tools

**Instant high-risk:** Photoshop, GIMP, Paint.NET, Pixlr, Canva, Inkscape, Adobe Illustrator, CorelDRAW, Photopea.

**Trusted tools:** Microsoft Word/Excel, LibreOffice, Google Docs, Apple Pages, Samsung/Apple/Huawei cameras.

### ExifTool Integration

Uses Phil Harvey's ExifTool when available for deep metadata extraction. Falls back to filesystem dates and Python libraries. Recommended: `sudo apt install libimage-exiftool-perl`

## Chain of Custody Protocol

### Four-Pillar Binding

The custody anchor binds four immutable elements:
1. **Device** — SHA-512 hash of hardware (IMEI + model + manufacturer)
2. **User** — Profile ID registered in the system
3. **Case** — Unique case identifier
4. **Time** — ISO 8601 UTC timestamp

### Anchor Computation

```
anchor = SHA-512(device_hash : profile_id : case_id : timestamp)
```

### Verification

Recompute the anchor from the same four inputs. If the hash matches, the chain is intact. If any element was altered, the hash differs — revealing tampering.

### Blockchain Anchoring

Custody anchors are submitted to OpenTimestamps for Bitcoin blockchain anchoring, creating an immutable, publicly verifiable timestamp that predates any dispute. See Constitution v6.0 — Section 8 (Verification Protocol).

## Security Considerations

### SHA-512 Throughout

All hashes use SHA-512, producing 128-character hex strings. Future-proof against known cryptanalytic attacks. Aligns with Nine-Brain Architecture verification protocol.

### No PII in QR Codes

QR codes contain only 8-character prefixes of hashes, never full identifiers. Full verification requires the Verum Global Foundation endpoint with authenticated access.

### Device Binding

Device serial numbers (IMEI) are never transmitted in plain text. Only the SHA-512 hash is stored and shared. The original serial number remains on the user's device only.

### Metadata Stripping Detection

Missing metadata is itself a signal. If a document has no creator tool information, the fraud detector assigns 15 risk points — because legitimate files almost always have metadata, and its absence suggests deliberate stripping.

### Article X Compliance

Identity data is used solely for legal truth-seeking and court proceedings. It is never sold, never used for surveillance, and never weaponized against vulnerable populations. Bound by Article X of the Verum Omnis Constitution — the Non-Weaponization Doctrine.
